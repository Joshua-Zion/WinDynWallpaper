# 修复动态壁纸无法获取窗口句柄 - 任务总结

## 问题
应用动态壁纸时报错 "无法获取桌面壁纸窗口句柄"。问题位于 `wallpaper-manager.ts` 的 `getWorkerWHwnd()` 方法。

## 根因分析

**PowerShell 子进程 Window Station 隔离**
- 原方案：通过 PowerShell 子进程调用 Win32 API 查找 WorkerW 窗口
- 失败原因：PowerShell 子进程运行在独立的 Window Station，即使 `EnumWindows` 能枚举到 Progman 和 WorkerW，`FindWindowEx` 和 `EnumChildWindows` 无法访问 Progman 的子窗口树
- 在 PowerShell 中，`FindWindowEx(Progman, null)` 返回 0，`FindWindowEx(Progman, ..., "SHELLDLL_DefView", null)` 也返回 0
- `GetShellWindow()` 在 PowerShell 中能找到 Progman，但无法枚举其子窗口

**关键发现（通过 edge-js C# 验证）**
- Electron 主进程运行在 **WinSta0/Default** 桌面（正确）
- 但 PowerShell 子进程无法访问 Progman 的子窗口层级
- 桌面结构：
  - Progman (h=1019C) 的直接子窗口：
    - SHELLDLL_DefView (h=101A0, visible) - 桌面图标
    - WorkerW (h=3B0672, visible) - 壁纸层

## 解决方案

**使用 edge-js 在主进程内直接调用 Win32 API**
- edge-js 编译 C# 代码在 Node.js 进程中运行，绕过 PowerShell Window Station 隔离
- C# 代码调用：
  1. `GetShellWindow()` 获取 Progman 句柄（可靠）
  2. `SendMessage(Progman, 0x052C, ...)` 触发创建 WorkerW
  3. `FindWindowEx(Progman, null)` 枚举 Progman 直接子窗口
  4. 查找可见的 WorkerW 返回其句柄

## 技术细节

### edge-js 用法注意
- `Invoke` 方法必须返回 `Task<object>`（不能用 `object`）
- `async Task<object> Invoke(object input)` = 正确签名
- C# 中 `out` 参数在 lambda 内需先声明变量再传递
- edge-js 编译后的 DLL 被 node_modules 缓存，每次 `edge.func()` 调用从缓存加载，无性能问题
